# 提高吞吐量方法整理

# 链下扩容

  • 闪电网络Lightning Network（有限）（减少放进区块的数据）

    闪电网络这一概念由Joseph Poon和Thaddeus Dryja在2015年提出。

    在有频繁交易的节点之间以保证金的方式构建一个预付款的池，只要不超过这个额度，所有的交易不在主链区块里记录，只有在结算的时候才发生一笔交易，从而减轻小额交易对主网的压力。平时自己维护一个账本，最后用BTC结算了再交给主链。闪电网络可以支持应用，相当于有一个侧链，结算的时候用BTC。

    优点：支付立即完成（几分之一秒内完成），实现微额支付（手续费是根据支付金额百分比付的）。

    缺点：中间节点故障会需要等待几小时甚至几天，对等节点失效会导致需要等待几小时才能关闭通道再花费资金，需要观察对等节点是否尝试花费一笔无效交易，无法线下交易（只有对方在线时交易才是可信任的），大额支付手续费高。

  • 雷电网络Raiden Network（有限）（减少放进区块的数据）

    雷电网络是一个链外扩展解决方案，用于在以太坊区块链上执行符合ERC20的令牌传输。这是比特币闪电网络的以太坊版本，支持近乎即时，低费用，可扩展和保护隐私的付款。

    优点：可扩展（参与者越多，雷电网络处理转账能力越高），更快（转账可以瞬间被确认），保护隐私（单笔转账不显示在总共享账本上），互操作性（支持所有遵循以太坊标准代币API标准的代币），更低的费用（雷电网络上的交易费用比区块链上的交易费用少7个数量级），微支付（更低的交易费用，使得雷电网络可以有效地进行微支付）

  • 侧链SideChain（有限）（减少放进区块的数据）

    也叫缠绕链，在技术上讲，就是缠绕结算链和应用链，使应用链数据和结算链数据无缝对接，承上启下。

    侧链主要功能可以有：结算链离链交易（类似微信及支付宝钱包之于银行账户，即闪电网络）、应用链的基础用户数据（包括用户账号、权限、代币账本）、应用链的运行承载（智能合约虚拟机、运行平台）。

    通过双向锚定Two-way Peg技术。

    以比特币上的侧链为例的流程：

    1. 把比特币转移到特意形成的地址上面。转移之后的比特币就被固定在这个地址上，你已经丧失了对其的控制权，实际上现在没有人对其拥有控制权。

    2. 一旦固定比特币的交易被确认，你就给侧链发送一条消息，消息要包含是你做了比特币转移的交易自己该比特币已经被固定住了。

    3. 一旦第二条区块链同意比特币的侧链，那按照预先的协议，会给你一定数量的该区块链上的代币，现在你对这些代币拥有控制权。

    4. 具体的效果来看确实像是比特币流转到了别的区块链上，实际上你拥有的比特币只是被固定住了，没有被销毁成为新的代币。

# 链上扩容

  • 区块大小扩容

    如果区块的大小过大，则会危害到比特币的安全模型，作为一种货币应用，这显然是不能令人接受的。

    随着区块变大，全网最后一个节点收到区块的时间也就越长，当恶意节点掌握足够算力，恶意节点在自己挖出的区块上继续挖矿且不公布广播，则恶意节点上没公布的区块长度，会大于全网区块长度；一旦恶意节点公布这些区块，则全网其它节点挖出的区块全部作废。

    区块设计过大，会威胁到比特币的安全。

  • 隔离见证（有限）（减少放进区块的数据）

    隔离见证 (Segregated Witness) 由比特币核心开发员Pieter Wuille在2015年12月提出。

    以BTC为例，隔离见证之前，数字签名占据全部交易数据的60-70%的空间，即隔离见证数据约为非隔离见证数据的2倍，使得区块大小不变的情况下，通过减少单笔交易的信息量，来容纳更多的交易。当所有的交易都采用隔离见证技术时，隔离见证大致相当于2MB的软分叉扩容。

    优点：扩容加速、为比特币的闪电网络部署打下基础（理论上闪电网络并不必要有隔离见证，但效果会打折扣）、解决比特币延展性攻击问题
