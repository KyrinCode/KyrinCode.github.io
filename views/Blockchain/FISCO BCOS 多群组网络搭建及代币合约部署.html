            FISCO BCOS 多群组网络搭建及代币合约部署

            Kyrin 2021-01-13 fiscobcos

            部署联盟链 FISCO BCOS 流程

            参考 https://fisco-bcos-documentation.readthedocs.io/zh_CN/latest/index.html

            搭建过程放在 https://github.com/KyrinCode/Fisco_Bcos_Setup 的 fiscobcos-sample0

            # 网络拓扑

            搭建单机、4 机构、3 群组、8 节点的网络拓扑：

            星形区块链组网如下：

            • agencyA：在 127.0.0.1 上有 2 个节点，同时属于 group1、group2、group3；
            • agencyB：在 127.0.0.1 上有 2 个节点，属于 group1；
            • agencyC：在 127.0.0.1 上有 2 个节点，属于 group2；
            • agencyD：在 127.0.0.1 上有 2 个节点，属于 group3。

            （在不同机器操作时，请将生成的对应IP的文件夹拷贝到对应机器启动，建链操作只需要执行一次！）

            # 网络配置

            创建工作目录。

            mkdir fisco && cd $_
            1

            获取 build_chain 脚本。

            curl -#LO https://github.com/FISCO-BCOS/FISCO-BCOS/releases/download/v2.7.1/build_chain.sh && chmod u+x build_chain.sh
            1

            生成区块链配置文件 ipconf。

            cat > ipconf << EOF
127.0.0.1:2 agencyA 1,2,3
127.0.0.1:2 agencyB 1
127.0.0.1:2 agencyC 2
127.0.0.1:2 agencyD 3
EOF

# ip:num：物理机 IP 以及物理机上的节点数目
# agency_name：机构名称
# group_list：节点所属的群组列表，不同群组以逗号分隔

            使用 build_chain 脚本构建区块链节点配置文件夹，这步会下载 fisco-bcos 的二进制文件，此时的 fisco/nodes/cert 中有各机构的证书文件，此时的 fisco/nodes/127.0.0.1/sdk 应为 agencyA 的 sdk 目录。

            # 根据配置文件生成区块链 需要保证机器的 30300~30301，20200~20201，8545~8546 端口没有被占用
# 其中 -p 选项指定起始端口，分别是 p2p_port,channel_port,jsonrpc_port
bash build_chain.sh -f ipconf -p 30300,20200,8545
            1
            2
            3

            （若生成的区块链节点属于不同物理机，需要将区块链节点拷贝到相应的物理机）

            启动节点。

            节点提供 start_all.sh 和 stop_all.sh 脚本启动和停止节点。

            cd fisco/nodes/127.0.0.1
bash start_all.sh # 启动节点
ps aux | grep fisco-bcos # 查看节点进程

            查看群组共识状态。

            # 节点正常共识打印 +++ 日志，+++ 日志字段含义：
# g:：群组 ID
# blkNum：Leader 节点产生的新区块高度；
# tx: 新区块中包含的交易数目；
# nodeIdx: 本节点索引；
# hash: 共识节点产生的最新区块哈希。

# 查看 node0 group1 是否正常共识
tail -f node0/log/* | grep "g:1.*++"

            # 控制台配置

            节点负责共识和记账，账户负责通过控制台通过 Java SDK 连到机构发送交易。

            获取控制台。

            cd fisco
curl -#LO https://github.com/FISCO-BCOS/console/releases/download/v2.7.1/download_console.sh && bash download_console.sh
            1
            2

            准备出 4 个机构的控制台。

            mv console consoleA
cp -r consoleA consoleB
cp -r consoleA consoleC
cp -r consoleA consoleD
            1
            2
            3
            4

            当前的 sdk 目录属于 agencyA。

            cd fisco/nodes/127.0.0.1
mv sdk sdk_agencyA
            1
            2

            为其余机构生成 sdk。

            获取证书生成脚本。

            curl -#LO https://raw.githubusercontent.com/FISCO-BCOS/FISCO-BCOS/master/tools/gen_node_cert.sh
            1

            通过指定机构私钥证书生成新 SDK，sdk 目录中存在机构新签发的证书和私钥。

            # -c指定机构证书及私钥所在路径
# -o输出到指定文件夹
bash gen_node_cert.sh -c ../cert/agencyB -o sdk_agencyB -s
bash gen_node_cert.sh -c ../cert/agencyC -o sdk_agencyC -s
bash gen_node_cert.sh -c ../cert/agencyD -o sdk_agencyD -s
            1
            2
            3
            4
            5

            将机构的 SDK 证书拷贝至控制台配置目录。

            cd fisco/consoleA
cp fisco/nodes/127.0.0.1/sdk_agencyA/* conf/

cd fisco/consoleB
cp fisco/nodes/127.0.0.1/sdk_agencyB/* conf/

cd fisco/consoleC
cp fisco/nodes/127.0.0.1/sdk_agencyC/* conf/

cd fisco/consoleD
cp fisco/nodes/127.0.0.1/sdk_agencyD/* conf/

            grep "channel_listen_port" fisco/nodes/127.0.0.1/node*/config.ini
            1

            根据 channel_listen_port 去修改控制台配置文件。

            cp fisco/consoleA/conf/config-example.toml fisco/consoleA/conf/config.toml

cd fisco/consoleB/conf
sed 's/20200/20202/g' config-example.toml | sed 's/20201/20203/g' > config.toml

cd fisco/consoleC/conf

cd fisco/consoleD/conf
sed 's/20200/20206/g' config-example.toml | sed 's/20201/20207/g' > config.toml

            # 部署代币合约

            将 tokenERC20.sol 和 TokenA.sol 放入 fisco/consoleA/contracts/solidity 和 fisco/consoleB/contracts/solidity 目录中。（机构 A 和机构 B 在群组 1 中）

            合约方法：

            name()
symbol()
decimals()
totalSupply()
balanceOf(address account)
transfer(address to, uint256 value)
burn(uint256 value)

            启动控制台。

            cd fisco/consoleA
bash start.sh
            1
            2

            控制台 consoleA 中部署合约。

            deploy TokenA
            1

            控制台 consoleB 中调用合约。

            call TokenA <contract address> name
