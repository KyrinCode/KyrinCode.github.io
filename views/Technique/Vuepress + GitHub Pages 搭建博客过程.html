            Vuepress + GitHub Pages 搭建博客过程

            Kyrin 2020-03-09 vuepress

            记录自己心血来潮搭博客的过程，涉及到的 node、npm、Vuepress、GitHub Pages 都是首次接触，亲自趟了些坑，加在一起前前后后两天搞完。

            # 安装插件

            # markdown-it-katex

            KaTeX 是一个支持 HTML 的轻量级的数学公式引擎，它由 Khan Academy 开发，采用 TeX 语法，快速且无需依赖库。

            先安装插件 markdown-it-katex：

            npm install markdown-it-katex
            1

            然后在 ./docs/.vuepress/config.js 文件中添加：

            module.exports = {
    ...
    markdown: {
        ...
    	extendMarkdown: md => {
     	md.use(require("markdown-it-katex"));
        }
    }
}

            然后有两种选择，第一种是在需要渲染 latex 公式的 md 文件开头加上：

            <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/KaTeX/0.5.1/katex.min.css">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/github-markdown-css/2.2.1/github-markdown.css"/>
            1
            2

            这样就需要每写一篇需要公式的文章都加一遍，不够优雅，于是还有第二种选择。在 ./docs/.vuepress/config.js 文件中添加如下两行：

            module.exports = {
    ...
    "head": [
        ...
        ["link", {"rel": "stylesheet", "href": "https://cdnjs.cloudflare.com/ajax/libs/KaTeX/0.5.1/katex.min.css"}],
        ["link", {"rel": "stylesheet", "href": "https://cdn.jsdelivr.net/github-markdown-css/2.2.1/github-markdown.css"}]
    ]
}

            # 参考

            一个日本人用 Vuepress 搭建的博客上，对一些扩展的安装过程解释得还挺详细的。

            # 设置GitHub Pages

            新建 Repository，命名为 <Username>.github.io，设置为 Public 然后 Create Repository。

            进入 Settings 里 GitHub Pages 中选择 Source 为 master（也在下面可以选择比较单调的自带 theme）。

            git clone https://github.com/<Username>/<Username>.github.io.git #在本地工作目录拉取仓库
            1

            可以将 .git 以外的所有文件删除。

            # 自定义设置

            图片替换

            在 ./node_modules/vuepress-theme-reco/images/ 下替换博客背景及个人头像。

            主页修改

            在 ./node_modules/vuepress-theme-reco/components/HomeBlog.vue 文件中，删除主页上的名字（重新搭建时会再次用到）。

            编译上传

            为了每次可以自动向远程博客发布新的内容，在 Workplace 目录内如此安排。维护两个目录分别是 <Username>_Blog 和 <Username>.github.io。

            <Username>_Blog 中用于本地编译运行 Vuepress，执行前面选择 Vuepress 模板流程。

            <Username>.github.io 是 git 的 Repository，仅远程同步 build 生成的静态文件。

            // 在./docs/.vuepress/config.js中添加dest，使build生成的静态文件存在<Username>/public
module.exports = {
	dest: 'public',
}
            1
            2
            3
            4

            编写工作流脚本 auto.sh，完成每次自动提交。

            #!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 将GitPages目录下除.git外所有内容放入oldversion文件夹
cd ../<Username>.github.io
mkdir oldversion
mv $(ls | grep -v "^\.git$" | grep -v "^oldversion$") oldversion
mv oldversion ../<Username>_Blog/
cd -

# 生成静态文件，目的位置在./docs/.vuepress/config.js中定义为./<Username>_Blog/public
npm run build

# 将build的静态文件放入仓库
mv public/* ../<Username>.github.io/

# 将附加内容放入仓库，可选
cp -r attach ../<Username>.github.io/

# 进入生成的文件夹
cd ../<Username>.github.io

# 提交
git add -A
git commit -m 'update webpages'
git push origin master

# 删除oldversion文件夹
cd -
trash oldversion
